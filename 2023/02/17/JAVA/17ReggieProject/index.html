存在的问题

数据库中的id值是19位的long型数据，发送给前端页面时，前端js损失了精度，只保证了前16位数字，所以修改时获取的id和数据库中的不一致。

解决办法是，服务端给页面响应数据时，将 long型的数据统一转为string类型。
