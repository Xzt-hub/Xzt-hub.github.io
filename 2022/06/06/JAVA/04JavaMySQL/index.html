数据库

  • 数据库：简称DB（DataBase），按照一定格式存储数据的一些文件的组合。存储数据的仓库，实际就是一堆文件，这些文件中存储了具有特定格式的数据。
  • 数据库管理系统：简称DBMS（DataBaseManagement），数据库管理系统是专门用来管理数据库中数据的，数据库管理系统可以对数据库当中的数据进行增删改查。
    • 常见的数据库管理系统：MySQL，Oracle，MS SqlServer，DB2, sybase等…
  • SQL：结构化查询语言，程序员需要学习SQL语句，程序员通过编写SQL语句，然后DBMS负责执行SQL语句，最终来完成对数据库中数据的增删改查操作。
    • SQL是一套标准，

三者之间的关系

​ DBMS —> 执行 ----> SQL —> 操作 ----> DB

查看MySQL服务

​ 计算机点击右键，选择管理，在选择服务和应用程序中的服务，找到mysql服务。默认是自动启动状态，只有在启动状态才能够使用。

SQL 分类⭐

  • DQL：数据查询语言，关键字：select
  • DML：数据操作语言，表中数据的增删改，delete、insert、update
  • DDL：数据定义语言，表结构的增删改create、drop、alter主要操作的是表结构
  • TCL：事务控制语言，事务提交commit，事物回滚rollback
  • DCL：数据控制语言，授权grant，撤销权限revoke…

数据库中最基本的单元是表：table

任何一张表都有行和列：

  • 行（row）：被称为数据/记录。
  • 列（column）：别成为字段，每个字段都有字段名，数据类型，约束等属性。
    • 字段名：是一个普通的名字，见名知意
    • 数据类型：字符串，数字，日期等
    • 约数：约束有很多，其中有唯一性约束。

DQL数据查询语言

简单查询

  • 查询一个字段
  • 查询两个字段或多个字段，使用,隔开
  • 查询所有字段，使用*进行查询时，效率低，可读性差
  • 给某个字段名起别名，使用as 关键字，其中as关键字可以省略。只是将显示的查询结果列名显示为别名。若别名中有空格或是中文则可以使用单引号将别名括起来
  • 使用数学表达式

条件查询

不是将表中所有数据都查出来，是查询出来符合条件的。

  • =等于条件
  • <>、!= 不等于
  • < ，<=，>，>=
  • between...and... 必须左小右大， 等同于 >= and <=
  • is null ，is not null，在数据库中 null不能使用=衡量。
  • and 并且
  • or或者

and 和 or 同时出现，有优先级问题，and优先级比or高

  • in，相当于多个or，不是区间，是个具体的值。not in
  • not 取非，通常使用于 is或in中
  • like， 模糊查询，%代表匹配任意个字符，_ 任意一个字符

排序

  • 关键字：order by 默认升序，降序关键字：desc，指顶升序asc
  • 两个字段或者多个字段排序
  • 根据字段的位置排序，不建议！！！
  • 当条件语句和排序共存时，需要先写条件语句再写排序语句

数据处理函数 / 单行处理函数

单行处理函数 的特点：一个输入对应一个输出

多行处理函数：多个输入对应一个输出

  • upper() 将该字段的所有字母都变为大写
  • lower 将该字段的所有字母都变为小写
  • substr(截取的字符串, 起始下标, 截取长度)取子串，起始下标从1开始。
  • length取长度
  • concat(str1, str2) 字符串拼接
  • trim() 去空格
  • round(小数, 保留的小数位)四舍五入，保留的小数位可以为负数，-1则代表保留到十位。
  • rand() 生成随机数，生成0 ~ 1的随机数
  • ifnull(数据，被当作哪个值) ，空处理函数，可以将null转换为一个具体的值。
  • case...when...then...when...then...else...end

分组函数 / 多行处理函数

输入多行，最终输出一行。分组函数在使用时必须先进行分组才能够使用。若没有分组则默认整张表为一组。

  • max、count、avg、min、sum

注意

  1. 分组函数自动忽略null，不是一个值，是什么也没有
  2. 分组函数中count(*)和count(具体字段)不同。
  3. 分组函数不能直接使用在where子句，因为where的执行顺序比group by高。参考下面的执行顺序

分组查询

可以将数据先进行分组，然后都每一组数据进行操作。

关键词：group by

关键字顺序：select...from...where...group by...order by...

执行顺序：from -> where -> group by -> select -> order by⭐

结论：在一条select 语句中，如果有group by语句的话，select后面只能跟：参加分组的字段，以及分组函数，其他的一律不能跟。

  • 联合分组
  • having

使用having可以对分完之后的数据进一步过滤。

注意：having不能单独使用，不能代替where，必须和group by联合使用。

优化策略：优先选择where

  • 总结：

关键字顺序：select...from...where...group by...having...order by... 只能按照这个顺序来

执行顺序：from -> where -> group by -> having -> select -> order by

连接查询

从一张表中单独查询称为单表查询。跨表查询称为链接查询。

根据连接方式分类

  • 内连接
    • 等值连接
    • 非等值连接
    • 自连接
  • 外连接
    • 左连接
    • 有连接
  • 全连接

当两张表进行连接查询没有任何条件限制的时候，最终查询结果条数，是两张表条数的乘积。这种现象称为 笛卡尔积现象。

表的连接次数越多，效率越低。则需要避免尽可能多的表连接

内连接之等值连接

table1 inner join table2 on 条件 内连接

内连接之非等值连接

连接条件是非等值关系

内连接之自连接

将一张表看作两张表，给一张表起两个别名。

外连接之右连接

关键词：right [outer] join...on

内连接的特点：将完全匹配上的连接条件查询出来。

外连接（右）：表示将join关键字右边的表看作主表，需要将其中的数据会全部查出来。两张表存在主次关系。

外连接之右连接

关键词：left [outer] join...on

表的创建

语法格式

表名建议以t_或者tb_开始，可读性强，见名知意。

字段名：见名知意。

数据类型：

  • vachar 可变长度字符串，会根据数据实际长度动态分配长度，最长255。
  • char 字符，不会根据数据实际长度分配长度，只会分配固定长度，会导致空间浪费
  • int 整型
  • bigint 等同于java中的long
  • float double
  • date 短日期
  • datetime 长日期
  • clob 字符大对象，最多可存储4G的字符串。超过255字符的都需要使用。
  • blob 二进制大对象。专门用来存储图片，声音，视频等流媒体数据。往blob类型的字段上插入数据时需要使用IO流才行。
