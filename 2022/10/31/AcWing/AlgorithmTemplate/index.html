并查集

  • p[i] 存放的是节点 i 的父节点
  • 判断两个节点是否属于一个集合，直接判断两个节点的祖宗节点是否相等即可。
  • find(x) 函数返回的就是x的祖宗节点，其中还进行了优化，将每一个节点都直接指向了祖宗节点。
  • 合并a、b两个节点，就是将a的祖宗节点指向b的祖宗节点即可。相当于给 a 的祖宗节点找了个爸😄。p[find(a)] = find(b)

堆

堆是一颗完全二叉树。

小根堆：每个节点的值都小于等于左右子节点的值，

大根堆：每个节点的值都大于等于左右子节点的值。

priority_queue 是STL中的堆，默认是大根堆，定义为小根堆的方式，priority_queue<int, vector<int>, greater<int>> q

插入新的元素，将数据插入数组末尾，然后用up函数依次交换。

  • heap[++ size] = x; up(size);

删除元素，将需要删除的元素和末尾元素交换，然后将最后一个元素删除，然后用down函数将交换的元素位置进行调整。

  • 删除最小值：heap[1] = heap[size]; size --; down(1);
  • 删除任意元素：heap[k] = heap[size]; size --; down(k); up(k);，其实down和up只会执行一个
  • 修改任意元素：heap[k] = x; down(k); up(k);

拓扑排序

只针对有向图，无向图不存在拓扑序列。本质就是宽度优先遍历的应用。

有向无环图 一定存在拓扑序列，有向无环图又被称为 拓朴图

算法步骤

  • 将所有入度为0的点入队，
  • 取出对头，枚举对头元素的所有出边t->j，删掉边t->j，d[j] -- 表示将节点 j 的入度减一
  • 如果d[j] == 0 则将 j 入队

时间复杂度：O(n + m) ，n表示点数，m表示边数

Dijkstra算法

朴素版dijkstra

算法步骤

  • 迭代n - 1次，因为起点已经加入了，所以只需要再找n - 1个点即可。
  • 在没有确定的节点中找距离起点值最小的节点。
  • 循环遍历用该节点更新起点到其他节点的距离。

时间复杂度: O(n ^ 2 + m), n 表示点数，m 表示边数

堆优化版dijkstra

算法步骤

和朴素版dijkstra算法类似，但是每次不需要找没确定的距离起点最小的节点。因为采用小根堆的方式存储，堆顶元素永远存储的是当前距离起点最近的节点。

  • 若该节点已经确定过了，则取出下一节点，直到取到既没有确定也是距离起点最小的节点为止。
  • 用该节点更新剩余没确定节点和起点间的距离

时间复杂度: O(mlogn), n 表示点数，m 表示边数

Prime算法

算法步骤

形似朴素版的dijkstra算法

  • 迭代 n 次（和dijkstra算法不同），因为没有起点需要加入，所以总共需要加入n个节点。
  • 每次在没确定的顶点集合中找到距离距离已确定节点距离最小的节点
  • 若该节点 不是第一个加入的节点 并且 距离已经确定的节点不是正无穷 ，则将该距离加入结果中。
  • 然后循环更新其他节点到 已经确定的节点集合 的最小距离

时间复杂度: O(n ^ 2 + m), n 表示点数，m 表示边数，适合稠密图

Kruskal算法

算法步骤

  • 先将所有的边按权值大小从小到大进行排序O(mlogm)
  • 枚举每条边，若 a 和 b 不在同一个集合，则将 a b加入同一个集合。需要使用到并查集

时间复杂度: O(mlogm), n 表示点数，m 表示边数，适合

二分图

二分图：可以将所有节点划分至两个集合，使得所有的边都是在集合之间，集合内部没有边。

性质：一个图是二分图当且仅当图中不含奇数环

染色体法辨别二分图

作用：判断一个无向图是否是一个二分图

算法步骤

  • 从节点1 遍历至节点n；如果第i个节点没有染色，则使用深度优先遍历队其及其所在的连通块进行染色。
  • 如果在染色过程中发现矛盾，则不是二分图

时间复杂度: O(n + m), n 表示点数，m 表示边数

匈牙利算法

作用：当给定一个二分图的两个集合，求集合中最大的匹配数

算法步骤

  • 求最大匹配数，依次枚举第一个集合中的每个点能否匹配第二个集合中的点
  • 循环遍历集合1节点能够匹配的集合2中的节点，若集合2中的某个节点没有被匹配 或者 匹配该节点的集合1中的节点可以匹配其他节点，则可以匹配。并且使用match[] 数组记录集合2节点匹配的集合1节点。
  • 注意：st 数组每次都需要初始化。是为了确保集合1中后面节点能够继续尝试已经匹配的集合2中的节点

时间复杂度: O(nm), n 表示点数，m 表示边数

素数

朴素筛法求素数

核心原理：当遍历到某一个数 i ，就将其所有倍数全部删掉，然后若当前这个数没有被删掉，则说明从2 ~ i - 1，没有其约数，则其为质数。

优化 ：每次删除时只需要将质数的倍数删除即可。

时间复杂度：O(n $\log(\log{n})$)

算法思想：辗转相除法。a是较大的数，b是较小的数，若b不为0，则让a赋值为b，b赋值为a % b，；若b为0，则直接输出a即可。

欧拉函数

欧拉函数定义

朴素版求欧拉函数，将N分解质因子，并求欧拉函数。

时间复杂度：O($\sqrt{n}$)

筛选法求欧拉函数

时间复杂度：O(${n}$)

代码解释：

  • 质数 i 的欧拉函数即为phi[i] = i - 1，1 ~ i - 1均与 i 互质，共i-1个。
  • phi[primes[j] * i] 分为两种情况：
    1. i % primes[j] == 0 时：primes[j] 是 i 的最小质因子，也是primes[j] * i 的最小质因子，因此1 - 1 / primes[j] 这一项在 phi[i]中计算过了，只需要将基数N修正为primes[j]倍即可，最终结果为phi[i] * primes[j]
    2. i % primes[j] != 0 时：primes[j] 不是 i 的质因子，只是primes[j] * i 的最小质因子，因此不仅需要将基数N修正为primes[j]倍，还需要跑补上1 - 1 / primes[j]这一项。则最终结果为phi[i] * (primes[j] - 1)
